# 先提问再执行

## 使用方式

**方式 A — 单独发送（推荐）**  
先发送下方 Prompt，等 Agent 提问并与你确认后，再补充具体任务。

**方式 B — 与任务合并**  
将 Prompt 放在具体需求**之前**，例如：

```
请你先向我提问，确保你完全理解了所有的需求之后再执行

我的任务是：重构用户模块的登录逻辑……
```

**方式 C — Cursor 引用**  
在对话中 `@prompts/meta/clarify-before-execute.md`，再描述你的任务。

## Prompt

```
请你先向我提问，确保你完全理解了所有的需求之后再执行
```

## 示例

**你发送：**

```
请你先向我提问，确保你完全理解了所有的需求之后再执行

我要给病历复印项目加一个支付宝退款状态查询接口。
```

**Agent 应先追问（而非直接写代码）：**

- 接口是新建还是改造现有路由？
- 需要对接哪套支付宝 API（当面付 / 小程序 / 其他）？
- 返回字段与错误码有无既定规范？
- 是否需要同步更新 API 文档或前端？

确认后再开始实现。

## 期望效果

- Agent **不假设**未说明的细节
- 多轮澄清后再执行，减少返工
- 适用于编码、写作、设计等各类任务
