# dingtalk-log 使用说明

将口语化的今日任务润色为可直接粘贴到钉钉的正式工作日志。

## 适用场景

- 填写钉钉「工作日志」「日报」「今日完成」
- 下班前快速整理当天任务条目
- 把零碎记录改成得体、专业的汇报用语

**触发关键词示例：** 写日志、日报、钉钉日志、今日工作、工作汇报、daily report

## 快速开始

1. 安装技能（若尚未安装）：

```bash
npx skills add xiongxianzhu/xskills --skill dingtalk-log -g -y
```

2. 在对话中逐条输入今日任务（每行一条，尽量简短）：

```
修复登录bug
优化首页加载
对接支付接口
```

3. 复制 Agent 输出的编号列表，粘贴到钉钉即可。

## 输入格式

| 方式 | 说明 | 示例 |
| --- | --- | --- |
| 换行逐条（推荐） | 每行一条任务 | 见上方快速开始 |
| 同一行多任务 | 用逗号、顿号或空格分隔，Agent 会先拆分 | `修 bug，开会，写文档` |
| 带项目前缀 | 保留项目名与模块名，便于润色 | `病历复印，修复支付宝退款错误提示` |

**建议：** 每条保留「做了什么 + 涉及模块/项目」，不必写完整句子。

## 输出格式

Agent 只输出**纯文本编号列表**，不含标题、日期或 Markdown：

```
1. 排查并修复登录模块的认证缺陷
2. 优化首页页面加载性能
3. 完成支付接口的联调对接工作
```

| 规则 | 说明 |
| --- | --- |
| 序号 | 使用 `1.` `2.` `3.` |
| 语气 | 书面、正式，无 emoji 与感叹号 |
| 内容 | 不编造数据、人名或未提及的技术细节 |
| 篇幅 | 1–3 条适当展开；4–6 条一句说清；7 条以上尽量精炼 |

## 完整示例

**输入：**

```
修了个支付页面的bug
跟产品对了下需求
优化了下列表查询性能
上线了新版本
```

**输出：**

```
1. 排查并修复支付页面的功能缺陷
2. 与产品团队进行需求评审，明确迭代方向
3. 优化列表查询性能，提升数据加载效率
4. 完成新版本发布上线
```

## 润色对照

| 用户输入 | 润色后 |
| --- | --- |
| 修复登录bug | 排查并修复登录模块的认证缺陷 |
| 改首页 | 优化首页页面加载性能 |
| 对接支付接口 | 完成支付接口的联调对接工作 |
| 开会 | 参与项目进度同步会议，明确下一阶段任务分工 |
| 写文档 | 梳理并输出技术文档 |
| code review | 完成代码评审，反馈优化建议 |

## 特殊情况

| 情况 | 处理方式 |
| --- | --- |
| 输入极其模糊（如「干活」「上班」） | 基于常见工作场景合理推断，保持通用表述 |
| 附带额外指令（如「重点写第一条」） | 按用户意图调整详略 |
| 同一行多个任务 | 先拆分，再逐条润色 |
