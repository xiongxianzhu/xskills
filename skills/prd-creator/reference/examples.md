# PRD 示例

## 示例 1：简化版（列表式）

**用户输入**：会员积分兑换，支持 Excel 批量导入会员，兑换商品区分自营和第三方

**输出**：

```markdown
# 会员积分兑换需求

时间：20260315

- 会员数据通过 Excel 文档导入，兑换商品由运营人工配置
- 会员记录不存在则自动创建
- 导入接口：Excel 导入会员信息，预设模板，一行一条会员记录
- 兑换下单接口：将会员与兑换商品关联，扣减积分，创建兑换订单
- 新增订单来源字段，用于区分自营/第三方对账
- 商品列表不展示第三方兑换商品，仅支持自营兑换
- 已兑换订单列表可展示第三方商品记录，不支持再次兑换
- 兑换商品新增渠道字段，区分自营或第三方
- 核销/发货流程与现有保持一致
```

## 示例 2：标准版（完整结构）

**用户输入**：做一个会议室预约功能

**输出**：

```markdown
# 会议室预约需求

**文档版本**：v1.0  
**创建日期**：20260315  

---

## 1. 项目概述

### 1.1. 背景与目标
- **业务背景**：员工需在线预约会议室，减少线下协调成本
- **核心目标**：支持员工选择会议室、时段完成预约
- **目标用户**：内部员工、行政管理员

### 1.2. 范围边界
- **范围内**：预约创建、取消、查询；会议室档期展示
- **范围外**：门禁联动、会议设备控制（本期不做）

---

## 2. 功能需求

### 2.1. 功能列表
| 序号 | 功能模块 | 描述 | 优先级 |
|------|----------|------|--------|
| 1 | 档期查询 | 按会议室/日期查询可预约时段 | P0 |
| 2 | 预约创建 | 选择时段提交预约 | P0 |
| 3 | 预约取消 | 用户取消未开始预约 | P0 |
| 4 | 预约列表 | 查询本人预约记录 | P0 |

### 2.2. 详细需求

#### 2.2.1 档期查询
- 应支持按会议室、日期筛选可预约时段
- 应展示剩余可预约数量
- 档期数据应与会议室系统同步，延迟不超过 5 分钟

#### 2.2.2 预约创建
- 用户选择时段后应校验身份（工号+姓名）
- 同一用户同一时段不得重复预约
- 预约成功后应推送站内通知

---

## 3. 非功能需求

- 响应时间：列表接口 < 2s
- 并发：支持 200 QPS

---

## 4. 约束与假设

- 依赖会议室系统提供档期查询接口
- 假设用户已通过企业 SSO 登录
```

## 示例 3：需求澄清对话

**用户**：帮我写个导出功能的 PRD

**Agent（应用技能后）**：需要先确认几项信息，以便写出可执行的 PRD：

1. **导出对象**：导出什么数据？（如：活动报名名单、问卷数据、设备台账）
2. **导出格式**：Excel、CSV、PDF？
3. **筛选条件**：支持按哪些条件筛选后再导出？
4. **权限**：谁可以导出？是否需要审批？
5. **数据量**：单次导出大概多少条？是否需要异步/分页？

请补充上述信息，或直接描述你的业务场景。
